# Vercel Deployment Guide - ORACLE AI System\n\n## Overview\n\nThis guide explains how to properly deploy the ORACLE AI System to Vercel and fix the 404 NOT_FOUND error.\n\n---\n\n## Problem: 404 NOT_FOUND Error\n\n**Error Details:**\n```\n404: NOT_FOUND\nCode: NOT_FOUND\nID: iad1::dqwxl-1779286278534-216393ab9d33\n```\n\n**Root Cause:**\nThe frontend is a Single Page Application (SPA) built with React/Vite. When Vercel tries to serve routes like `/integrated-control-center`, it looks for a matching file, doesn't find it, and returns 404. The SPA needs all routes to be rewritten to `index.html` so React Router can handle client-side routing.\n\n---\n\n## Solution: Vercel Configuration\n\n### Step 1: Verify `vercel.json` is in place\n\nThe `vercel.json` file has been created in `/Code-Feature-Evolve/vercel.json` with the following key configurations:\n\n```json\n{\n  \"version\": 2,\n  \"buildCommand\": \"cd artifacts/oracle && pnpm install && pnpm build\",\n  \"outputDirectory\": \"artifacts/oracle/dist\",\n  \"rewrites\": [\n    {\n      \"source\": \"/(.*)\",\n      \"destination\": \"/index.html\"\n    }\n  ]\n}\n```\n\n**What this does:**\n- **buildCommand**: Tells Vercel to build the Oracle frontend specifically\n- **outputDirectory**: Points to the compiled React app\n- **rewrites**: Redirects all routes to `index.html` so React Router handles them\n\n### Step 2: Update Vercel Project Settings\n\n1. **Go to Vercel Dashboard** → Your Project → Settings\n\n2. **Build & Development Settings:**\n   - **Framework Preset:** Vite\n   - **Build Command:** `cd Code-Feature-Evolve/artifacts/oracle && pnpm install && pnpm build`\n   - **Output Directory:** `Code-Feature-Evolve/artifacts/oracle/dist`\n   - **Install Command:** `pnpm install --recursive`\n\n3. **Root Directory:** Leave blank (or set to `Code-Feature-Evolve` if needed)\n\n4. **Environment Variables:**\n   ```\n   NODE_ENV=production\n   VITE_API_URL=https://your-api-domain.com\n   ```\n\n5. **Click \"Save\"**\n\n### Step 3: Redeploy\n\n1. Go to **Deployments** tab\n2. Click **Redeploy** on the latest deployment\n3. Or push a new commit to trigger auto-deploy:\n   ```bash\n   git add .\n   git commit -m \"Fix: Vercel deployment configuration\"\n   git push origin main\n   ```\n\n---\n\n## Vercel Configuration File Breakdown\n\n### Rewrites (SPA Routing)\n```json\n\"rewrites\": [\n  {\n    \"source\": \"/(.*)\",\n    \"destination\": \"/index.html\"\n  }\n]\n```\nThis ensures all routes are served by React Router.\n\n### Headers (Caching & Performance)\n```json\n\"headers\": [\n  {\n    \"source\": \"/(.*)\\.js\",\n    \"headers\": [\n      {\n        \"key\": \"Cache-Control\",\n        \"value\": \"public, max-age=31536000, immutable\"\n      }\n    ]\n  }\n]\n```\nThis caches static assets for 1 year (optimal for performance).\n\n### Redirects (URL Shortcuts)\n```json\n\"redirects\": [\n  {\n    \"source\": \"/oracle\",\n    \"destination\": \"/\",\n    \"permanent\": false\n  }\n]\n```\nThis allows friendly URLs to redirect to the main app.\n\n---\n\n## Testing Locally Before Deployment\n\n### 1.
