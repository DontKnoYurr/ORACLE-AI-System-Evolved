# Testing and Debugging Guide\n\n**Author:** Manus AI\n\n## 1. Overview\n\nThis guide provides comprehensive instructions for testing, debugging, and previewing the expanded AI system. The system has been designed with rigorous quality assurance in mind, featuring multiple layers of testing from unit tests to end-to-end integration tests.\n\n## 2. Testing Architecture\n\n### 2.1 Unit Tests\n\nUnit tests validate individual components in isolation:\n\n*   **Optimizer Module Tests** (`optimizers.test.ts`): Tests Adam, RMSprop, gradient clipping, learning rate scheduling, and anomaly detection.\n*   **Synthetic Generator Tests** (`synthetic-generator.test.ts`): Validates synthetic data generation across all interaction types.\n*   **Event Bus Tests** (`event-bus.test.ts`): Tests pub/sub functionality, filtering, history, and metrics.\n\n### 2.2 Integration Tests\n\nIntegration tests validate component interactions:\n\n*   **System Integration Tests** (`integration.test.ts`): Tests the complete pipeline from event publishing through training to metrics updates.\n*   **End-to-End Training Pipeline**: Validates the entire training workflow with event tracking.\n*   **Error Handling and Recovery**: Ensures graceful error handling across components.\n\n### 2.3 Test Coverage\n\nThe system includes over 100 unit tests and 20+ integration tests covering:\n\n*   Core algorithm correctness\n*   Edge cases and boundary conditions\n*   Concurrent operations\n*   Error scenarios and recovery\n*   Performance and scalability\n*   Data quality and consistency\n\n## 3. Running Tests\n\n### 3.1 Unit Tests\n\n```bash\n# Run all unit tests\npnpm test\n\n# Run specific test file\npnpm test optimizers.test.ts\n\n# Run tests in watch mode\npnpm test --watch\n\n# Run tests with coverage\npnpm test --coverage\n```\n\n### 3.2 Integration Tests\n\n```bash\n# Run integration tests\npnpm test integration.test.ts\n\n# Run integration tests with verbose output\npnpm test integration.test.ts --verbose\n```\n\n## 4. Debugging Techniques\n\n### 4.1 Event Bus Debugging\n\nThe event bus provides comprehensive debugging capabilities:\n\n```typescript\n// Get event metrics\nconst metrics = eventBus.getMetrics();\nconsole.log('Total events:', metrics.totalEvents);\nconsole.log('Events by type:', metrics.eventsByType);\n\n// Get event history\nconst history = eventBus.getHistory({\n  type: 'training:cycle:complete',\n  limit: 10,\n});\n\n// Get channel statistics\nconst stats = eventBus.getChannelStats('training:cycle:complete');\nconsole.log('Average processing time:', stats?.averageTime);\n```\n\n### 4.2 Training System Debugging\n\nMonitor training progress with detailed metrics:\n\n```typescript\nconst session = await createTrainingSession({\n  epochs: 50,\n  optimizer: 'adam',\n});\n\n// After each epoch\nconst summary = getSessionMetricsSummary(session);\nconsole.log('Loss improvement:', summary?.lossImprovement);\nconsole.log('Gradient norms:', summary?.maxGradientNorm);\nconsole.log('Anomalies detected:', summary?.anomalyCount);\n```\n\n### 4.3 SSE Connection Debugging\n\nMonitor SSE client connections:\n\n```typescript\n// Get all connected clients\nconst clients = sseManager.getAllClientsInfo();\nconsole.log('Connected clients:', clients.length);\n\n// Get statistics\nconst stats = sseManager.getStatistics();\nconsole.log('Total messages:', stats.totalMessages);\nconsole.log('Channel subscriptions:', stats.channelCounts);\n```\n\n## 5. Common Issues and Solutions\n\n### 5.1 Training Loss Not Decreasing\n\n**Symptoms:** Training loss remains constant or increases.\n\n**Debugging Steps:**\n\n1. Check learning rate: `console.log(metrics.learningRate)`\n2. Verify gradient computation: `console.log(gradientNorms.l2Norm)`\n3. Check for gradient anomalies: `detectGradientAnomalies(gradients)`\n4. Verify training data: Ensure interactions have appropriate training weights\n\n**Solutions:**\n\n*   Increase learning rate if gradients are very small\n*   Use gradient clipping if gradients are exploding\n*   Verify interaction data quality and distribution\n*   Try different optimizer (Adam, RMSprop, SGD)\n\n### 5.2 SSE Connection Drops\n\n**Symptoms:** Live updates stop appearing, connection shows as disconnected.\n\n**Debugging Steps:**\n\n1. Check browser console for errors\n2. Verify server is running: `curl http://localhost:3000/api/health`\n3. Check SSE endpoint: `curl http://localhost:3000/api/events`\n4. Monitor network tab for connection status\n\n**Solutions:**\n\n*   Restart SSE server\n*   Check firewall/proxy settings\n*   Verify CORS headers are set correctly\n*   Increase heartbeat interval if connection is unstable\n\n### 5.3 High Memory Usage\n\n**Symptoms:** Application becomes slow, memory usage increases over time.\n\n**Debugging Steps:**\n\n1. Check event history size: `eventBus.getHistory().length`\n2. Monitor SSE client count: `sseManager.getClientCount()`\n3. Check training session count: `activeSessions.size`\n\n**Solutions:**\n\n*   Reduce event history retention: Modify `maxHistorySize` in event bus\n*   Cleanup dead SSE connections: `sseManager.cleanupDeadConnections()`\n*   Complete training sessions: `completeTrainingSession(sessionId)`\n*   Implement periodic garbage collection\n\n## 6.
